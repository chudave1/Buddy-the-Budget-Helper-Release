TESTING INSTRUCTIONS — Buddy the Budget Helper

NO USER LOGIN REQUIRED
This app does not use usernames, passwords, Microsoft accounts, or any
publisher-hosted login. There is no test account to create. All data is stored
locally in a SQLite database. The app works fully offline. There is no sign-on
screen. Administrator rights are not required.

IMPORTANT: "Accounts" in Setup means bank/credit card definitions (e.g.,
"Checking", "Citibank Visa") for tracking spending. It does NOT mean user
authentication, and the app does NOT connect to any bank website.

FIRST LAUNCH (required)
1. Launch the app on a clean install (no prior version).
2. When "Welcome to Buddy the Budget Helper!" appears, click Yes to load the
   sample database.
3. Wait for "Loading Sample Database" to finish (Steps 1–5). Do not close the app.
4. When "Sample Database Loaded" appears, click OK.
5. The main window should show budget data for the prior and current calendar
   years and a populated category grid.

QUICK TEST (about 5 minutes)
Complete First Launch steps 1–5 above.

Optional — dashboards:
1. Menu: Analysis → Dashboards
2. Leave "Actual versus Budget" selected (default).
3. Choose any category from the Category dropdown.
4. Click Show Dashboard — a chart window should open with data.

FEATURES TO VERIFY
- Main budget window (change year in the dropdown)
- Analysis dashboards (steps above)
- Help opens documentation in the browser (only network use)

SAMPLE DATA LIMITATIONS
- The main window Details button does not show transaction-level detail for
  sample data (summary grid data only).
- Summary data covers the prior calendar year and the current year through
  two months before the test date.

PRIVACY
No personal data is collected or transmitted. See in-app Help or:
